CISCO ROUTING PROTOCOLS:

I - IS - IS (CCIE Level)
M - MPLS (Multiprotocol Layer Swtiching) (CCNP Level) (Label VIP)  -- Expressway concept
---------------------------------
B - BGP (Border Gateway Protocol) (CCNP Level, mentioned sa CCNA Exams)
O - OSPF (Open shortest Path first)
D - D'EIGRP (Enhancep Interior Gateway Protocol) (Cisco Proprietary)
-------------------------------- 
S - Static
S*- Default Static

1. PING-DIKIT

- also known for manual routing
- applying static routes requires 2 way to configure

recipe: ip route 
	<not directly connected Network IP>
	<not directly connected>
	<next hop host ip>

DYNAMIC ROUTING (EIGRP, OSPF, BGP)
- It only requires (advertise) directly connected networks
- Any discovered routes will do adjacency for connectivity

EIGRP Routing
- Cisco Propriety
- Most efficient routing protocol

recipe: 
     router eigrp _ (Autonomous System Number)
	network <directly connected network IP>
	no auto-summary
     end


 
@D1
conf t
  router eigrp 100
     no auto-summary
     network 10.1.4.4
     network 10.2.1.0
     network  10.2.2.0
     network  192.168.1.128
  end

OSPF Routing
- Advertise by AREA (Area 0 is the backbone)
- OSPF States: Down > Init > 2 - Way > EXSTART -> Exchange > Loading > Full

   recipe:
     router ospf # (process id)
     router -id (#.#.#.#)
     network  (directly connected NET IP) (Wildcard mask)



@R4
conf t
router ospf 1
   router-id 4.4.4.4
   network 4.4.4.4 0.0.0.0 area 34
   network 10.1.1.8 0.0.0.3 area 34
end


@R3
conf t
router ospf 1
   router-id 3.3.3.3
   network 3.3.3.3 0.0.0.0 area 34
   network 10.1.1.4 0.0.0.3 area 0
   network 10.1.1.8 0.0.0.3 area 34
end



@R2
conf t
router ospf 1
   router-id 2.2.2.2
   network 2.2.2.2 0.0.0.0 area 0
   network 10.1.1.0 0.0.0.3 area 12
   network 10.1.1.4 0.0.0.3 area 0
end



@R1
conf t
router ospf 1
   router-id 1.1.1.1
   network  1.1.1.1 0.0.0.0 area 12
   network 10.1.1.0 0.0.0.3 area 12

BGP Routing
- Border Gateway Protocol
- Mostly Used for WAN
- Mostly used by the ISP or Large Corporations



recipe:
    router bgp <AS No.>
        neighbor <direct IP of neighboring router> remote-as <remote AS No.>
        :
        :
        network <directly connected NET.IP to neighbor> mask <subnet mask>
        :
        :
    end



@R1
conf t
 router bgp 1
 bgp log-neighbor-changes
  neighbor 208.8.8.4 remote-as 45
  neighbor 209.9.9.3 remote-as 3
  neighbor 207.7.7.2 remote-as 2
  network 208.8.8.0 mask 255.255.255.0
  network 209.9.9.0 mask 255.255.255.0
  network 207.7.7.0 mask 255.255.255.0
  network 10.1.1.0 mask 255.255.255.252
end

sh ip bgp sum

ROUTE REDISTRIBUTION
(by default, different routing protocols cannot get along with each other)

@R1 (for BGP + OSPF)
conf t
router ospf 1
 default-information originate always
end


@R4 (for OSPF + EIGRP)
!pasok EIGRP to OSPF
conf t
router ospf 1
  redistribute eigrp 100 subnets
exit

!pasok OSPF to EIGRP
router eigrp 100
redistribute ospf 1 metric 1544 2000 255 1 1500
end
